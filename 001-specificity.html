Specificity


The CSS that is added in an html document can be added in different categories of tags.

When a css rule is applied in a particular tag, then that means that all the tags in the document are affected by the CSS rule.

Therefore, if you would add a particular width property like: div {width: 150px;},
that would mean that all the div tags in the document would have a width of 150 pixels.


As a developer, you have the option of being more specific on which tags should get the settings you have defined.

You can be more specific on which selectors get the properties you are working on by using:

  • A class property in the event that you want some of the same tags to be styled in a particular way but you do not want all the tags with the same styling.
  • An id property in case you want only one specific tag to be styled in a particular way but you do not want other tags to have the same styling.

There are several occasions where tags have more than one classes and more than one ids applied to them.

That is because we might be using different classes to accomplish different, overlapping or supplementing styling formats.

In order to be able to define the styling of tags in a priority base, CSS uses a specificity weight.

In short, specificity means that ids have more weight than classes, and classes have more weight than tags.


For example, let's assume you have an h2 tag that has the following structure:

<h2 class="redBoldClass" id="greenUnderlineId">

And also let's assume that you have the following CSS code (dot is for classes, # is for ids)

h2 {
color: yellow;
font-size: 30px;
}

.redBoldClass {
color: red;
font-size: 25px;
font-weight: bold;
}

#greenUnderlineId {
color: green;
font-size: 35px;
text-decoration: underline
}

The particular CSS tag would end up having a green color, a font size of 35px and a text decoration of underline.

That is because the ids have a greater weight in specificity.

On the other hand, all the other h2 tags that have the same class will inherit the properties from the .redBoldClass, while the h2 tags that do not have and class or id attached to them, will inherit the properties from the CSS applied in the h2 tag.

  • Where needed, the different tags were given class and id names.
